The Same Order in Four Formats

A simplified order: id="ord00042", restaurant_id="rest0001", status=5 (EnRoute), tip_cents=500, created_at=1700000000.0, platform_txn_id=9007199254740993

Python Objects -- Scattered Heap Allocations

Each field is a separate Python object on the heap. The Order object holds references (pointers) to each field. Accessing any field requires following a pointer and loading a separate cache line.

Total: ~800+ bytes of heap allocations (PyObject headers + data + reference counts)

Protobuf -- Compact Tagged Fields

Fields are packed sequentially with tag+type prefixes. Integers use varint encoding (small values = fewer bytes). Must parse ALL fields sequentially to find any one field.

Total: ~65 bytes. Compact, but requires full parse to read any field.

FlatBuffers -- VTable + Data (Back-to-Front)

Root offset points to table. Table starts with soffset to vtable. VTable maps field slots to byte offsets. Strings stored separately with length prefix.

Total: ~120 bytes. Zero-copy read, but vtable indirection for each field access.

Cap'n Proto -- Word-Aligned Data + Pointers (Front-to-Back)

Segment table, then root pointer, then struct (data section + pointer section), then text content. Everything word-aligned. No vtable needed -- field positions are fixed by schema.

Total: ~104 bytes. Zero-copy read AND write. No indirection -- field offset is computed directly.

64-Bit Pointer Anatomy

Every Cap'n Proto pointer packs type, offset, and size information into a single 8-byte word. Hover over each bit range to see what it encodes.

Hover over a bit range to see its meaning.

Example: Struct pointer to Order at offset=0 words, data=4 words, ptrs=3 words

0x 0003 0004 0000 0000

Bits 0-1: type=0 (struct) | Bits 2-31: offset=0 | Bits 32-47: data=4 | Bits 48-63: ptrs=3

Hover over a bit range to see its meaning.

Example: List pointer for text "ord00042\0" (9 bytes) at offset=3 words

0x 0000 0049 0000 000d

Bits 0-1: type=1 (list) | Bits 2-31: offset=3 | Bits 32-34: elem_size=2 (byte) | Bits 35-63: count=9
