3. Memory Allocations per Decode

Object allocations when reading fields from a serialized buffer. Fewer allocations = less GC pressure = stable p99 latency.

Timing Comparison (Python, ns/op)

Protobuf: scan for 1 int
~800 ns
FlatBuffers: read 1 int
~350 ns (2.3x faster)
Protobuf: scan for 2 fields
~1500 ns
FlatBuffers: read 2 fields
~650 ns (2.3x faster)

In C++/Rust, the FlatBuffers advantage is 10-100x because field reads compile to single pointer dereferences. Python's struct.unpack overhead narrows the gap, but the architectural advantage is clear.
